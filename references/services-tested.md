# Services tested

This file records the real outcomes from the sample YouTube video:

- URL: `https://www.youtube.com/watch?v=FrD5I3aGibc`
- Title: `Use Office 365 Connectors in Microsoft Teams`
- Channel: `Microsoft 365 Developer`

## Local pipeline

The local pipeline now works for this video:

- `yt-dlp` downloaded the MP4 successfully
- `yt-dlp` downloaded auto subtitles successfully
- `faster-whisper` transcribed the downloaded MP4 locally
- OpenCV extracted real frames from the downloaded MP4

This is the preferred path because it produces actual frame images and an actual transcript file, instead of only service diagnostics.

## Transcript services

### transcript-api.com

Public web flow tested:

`https://transcript-api.com/web/transcript?video_url=https%3A%2F%2Fwww.youtube.com%2Fwatch%3Fv%3DFrD5I3aGibc`

Observed result:

```json
{"detail":"Transcript not found for video: FrD5I3aGibc"}
```

API v2 endpoint exists but requires `Authorization` header, so it is not a no-auth fallback.

### 2outube

Public endpoint tested:

`https://2outube.com/api/transcript?v=FrD5I3aGibc`

Observed result:

```json
{"videoId":"FrD5I3aGibc","transcript":[],"available":false,"failureReason":"video_unavailable","failureMessage":"The page needs to be reloaded."}
```

Observed on another attempt:

```json
{"videoId":"FrD5I3aGibc","transcript":[],"available":false,"failureReason":"bot_detection","failureMessage":"YouTube is temporarily blocking this request. Try again in a few minutes, or paste the URL directly."}
```

Treat 2outube as a useful probe, but not an authoritative source of absence.

### Supadata

Page reachable:

`https://supadata.ai/youtube-transcript`

Observed result:

- service page reachable
- page claims free transcript extraction and AI fallback
- no no-auth public endpoint was validated in CLI

Treat it as a browser/manual fallback, not a proven direct CLI endpoint.

## YouTube HTML fallback

### Captions

Watch HTML contained `captionTracks` for this video.

Observed track:

- language: `en`
- type: autogenerated (`kind: "asr"`)

Important nuance:

- `captionTracks` in HTML proves captions are exposed by YouTube metadata
- direct timedtext fetch from CLI still returned `200` with empty body in this environment

So the correct conclusion is:

- captions exist
- direct transcript body fetch was blocked or empty from this client

### Storyboards / prints

Watch HTML contained `playerStoryboardSpecRenderer.spec`.

That proves YouTube exposed storyboard sheets for multiple frame prints.

Observed behavior:

- direct guessed `i.ytimg.com/sb/...` fetch returned `403 Forbidden`

So the correct conclusion is:

- storyboard print source exists
- direct sheet download was bot-blocked in this environment
